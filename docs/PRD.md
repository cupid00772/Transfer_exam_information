# 產品需求文件 (PRD)：實時轉學考資訊系統

## 1. 文件資訊 (Document Info)
- **文件狀態:** 草稿 (Draft)
- **作者:** AI 產品經理 (Antigravity)
- **建立日期:** 2026-04-22
- **目標平台:** Web (FastAPI 後端 + HTML 前端)

## 2. 產品概述 (Product Overview)
### 2.1 背景與問題 (Background & Problem)
每年有大量的大學生有轉學需求，但各大學的轉學考簡章發布時間不一，且資訊散落於各校教務處網站。考生需要耗費大量時間反覆查詢簡章公告、報名時程與資格限制，容易因為資訊落差而錯過報名期限或準備方向。
### 2.2 目標與願景 (Goal & Vision)
打造一個一站式的「實時轉學考資訊系統」，提供最新、最準確的轉學考資訊。幫助考生快速查詢各校系簡章、報名時程、資格及限制，成為準備轉學考的最佳資訊聚合平台。未來甚至可結合 AI 提供自然語言問答，成為考生的專屬顧問。

## 3. 目標用戶 (Target Audience)
### 3.1 使用者輪廓 (User Personas)
- **主要使用者 (轉學考生):** 準備參與今年或明年轉學考的大學生或專科畢業生，需要隨時掌握目標校系的報名與考試動態。
- **次要使用者 (補習班業者/家長):** 需要協助考生收集資訊、規劃考試策略、確認報名資格的人員。

## 4. 使用案例與使用者故事 (Use Cases & User Stories)
### 4.1 情境描述 (Scenarios)
使用者進入系統後，輸入目標學校（例如：台灣大學）或科系（例如：資訊工程學系），系統即時調出最新的轉學考資訊，條列式呈現：簡章連結、報名起訖時間、考試日期、報名資格與限制。使用者不需再親自到各校網站大海撈針，一眼就能確認自己是否符合報名資格。

### 4.2 使用者故事 (User Stories)
- 身為一位轉學考生，我想要「輸入學校或科系名稱」，這樣我才能「快速找到目標校系的轉學考資訊」。
- 身為一位轉學考生，我想要「直接看到報名時程與考試日期」，這樣我才能「避免錯過報名時間並安排讀書進度」。
- 身為一位轉學考生，我想要「清楚看到報名資格與特殊限制」，這樣我才能「確認自己是否符合該校系的報考規定」。
- 身為一位轉學考生，我想要「獲取官方簡章的原始 PDF 連結」，這樣我才能「下載並詳細閱讀官方規定的細節」。

## 5. 功能需求 (Functional Requirements)
### 5.1 核心功能列表 (Feature List)
- **1. 資訊查詢與過濾功能:** 
  - 支援以「學校名稱」、「科系名稱」為關鍵字進行搜尋。
- **2. 實時資訊獲取 (爬蟲模組/外部API):** 
  - 建立排程程式或即時爬蟲，定期從各大專院校教務處網站擷取最新的簡章公告。
- **3. 資訊結構化呈現介面:** 
  - 將各校資訊統整成統一的卡片或表格，呈現以下欄位：
    - 學校與科系名稱
    - 報名時程 (報名開始與結束日期)
    - 考試日期
    - 招生名額
    - 報名資格與特殊限制
    - 官方簡章連結
- **4. AI 對話助手 (可選/進階):** 
  - 允許使用者用自然語言提問（如：「今年成大資工轉學考什麼時候報名？有什麼限制？」），系統透過檢索資料庫來回答。

## 6. 非功能需求 (Non-Functional Requirements)
- **即時性與準確性:** 系統提供的資訊必須盡可能與官方公告同步，並標示「最後更新時間」。
- **效能:** 搜尋回應時間應低於 2 秒。
- **響應式設計:** 前端介面需支援手機版，方便考生隨時隨地透過手機查詢。
- **容錯性:** 爬蟲模組需具備錯誤重試與例外處理機制，避免因單一學校網站改版而導致系統崩潰。

## 7. 系統與相依性 (System & Dependencies)
- **前端技術:** HTML / CSS / Vanilla JavaScript (響應式設計)
- **後端技術:** Python + FastAPI
- **資料庫:** SQLite (儲存各校系簡章資訊與時程)
- **外部整合:**
  - 網頁爬蟲套件 (如 `BeautifulSoup4`, `Selenium` 或 `Playwright`) 用於自動化獲取資料。
  - (選配) LLM API (如 Google Gemini API) 用於自然語言解析與回答。

## 8. 成功指標 (Success Metrics)
- **查詢成功率:** 使用者進行查詢時，能正確返回最新簡章資訊的比例。
- **資料覆蓋率:** 系統能涵蓋台灣主要大專院校轉學考資訊的比例。
- **回應速度:** API 平均回應時間。
